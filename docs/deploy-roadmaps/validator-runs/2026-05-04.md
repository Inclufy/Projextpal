## Cross-cutting findings

- RAID-log is nu volledig dekkend: Risk + Assumption + Issue + Dependency (`charater.Dependency`) — geen losse Dependency-stub op Project nodig zolang `charater.Dependency` aanvaard wordt.
- Attachments-pattern is consistent toegepast (M2M → `projects.Upload`, `related_name="+"`) op 6 risk/governance-modellen — goed; pattern kan worden uitgerold naar `Decision` als toekomstige uitbreiding.
- `LessonLearned` heeft `applicable_to` (project/portfolio/organisation) — uitstekend voor MSP/portfolio-rolling-up.
- **`DefinitionOfDone` bestaat drie keer** (projects, scrum, agile). Overweeg consolidatie: één canonical model in `projects` en re-use vanuit scrum/agile via `OneToOneField` of proxy. Risico: data-divergentie tussen de drie. **Niet blocking voor deze deploy** — maar registreer als refactor-ticket.
- `governance.Decision` dekt PMBOK Decision Register goed (status, rationale, decision-date). Geen extra model nodig.

## Drift / regressions

- **Geen regressies geconstateerd**. Bestaande modellen (Risk, ManualMitigation, StatusReport, Meeting) behouden alle eerder velden — `attachments` is enkel additief (`blank=True`, geen NOT-NULL constraint).
- Drie parallelle DefinitionOfDone-implementaties is geen drift maar wel toekomstige technical-debt vlag — registreer als refactor-ticket, niet als coverage-gap.
- `related_name="+"` op alle attachments-M2M voorkomt reverse-accessor-vervuiling op `Upload` — bewust en correct.
