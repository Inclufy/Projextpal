## Governance/structure violations (highest priority) ‚ÄĒ "is there any DMAIC structure?"
**Yes ‚ÄĒ there is a DMAIC structure, and it is Green's.** `LSSBlackTask.phase` is an FK to
`lss_green.DMAICPhase` (`models.py:172-174`); the Black Phases/Tasks/Timeline/Overview pages all
read `/api/v1/lss-black/projects/<id>/dmaic-phases/`, a viewset re-imported from
`lss_green.views.DMAICPhaseViewSet` (`urls.py:10`). Black does **not** float free. **But:**

1. **‚ĚĆ Tollgate is not a gate.** `sixsigma.TollgateReview` (`sixsigma/models.py:1046`, has
   `status`/`approved`/`approved_by`) is a *record*. `DMAICPhase` completion is a bare PATCH with
   no tollgate precondition (`LSSBlackPhases.tsx:80-85` PATCHes `status:"completed"` directly). A
   Black Belt can close Analyze with no passed tollgate and no MSA. **No enforced phase gating.**
2. **‚ĚĆ Advanced tools float free of the spine.** `HypothesisTest`, `DesignOfExperiment`, `SPCChart`,
   `ControlPlan` (`lss_black/models.py:6,51,87,110`) have **only** `project` + `responsible` FKs ‚ÄĒ
   no `phase` FK, no link to a root cause, a metric, a Solution, or a tollgate deliverable. A DOE
   result cannot point at the Improve solution it justifies; a hypothesis test cannot attach to the
   `sixsigma.FishboneCause`/`Solution.addresses_root_cause` (`sixsigma/models.py:617`) it confirms.
3. **‚ö†ÔłŹ Duplicated model surface (spec safety-rule 4 violation risk).** `lss_black.HypothesisTest`
   (`models.py:6`) duplicates `sixsigma.HypothesisTest` (`sixsigma/models.py:482`, with
   `regression`/`correlation` types + `conclusion` choices). `lss_black.SPCChart`/`ControlPlan`
   duplicate the Control group's `control/spc` + `control/control-plan` (`AppSidebar.tsx:513-514`,
   backed by `sixsigma`). The Black sidebar exposes **two** "Hypothesis Tests" entries pointing at
   **two different backends**: `AppSidebar.tsx:449` (‚Üí lss-black) and `:492` (‚Üí sixsigma/Analyze).
   Confusing, divergent data, no migration path.
4. **‚ö†ÔłŹ No financial sign-off wired to Black work.** `sixsigma` has `estimated_savings`,
   `expected_savings`, and `ProjectClosure.projected/realized_savings` + `roi` property
   (`sixsigma/models.py:116,609,1142-1202`), but no `lss_black` artifact references benefits, and the
   Black Overview shows generic finance `cost-summary` (`LSSBlackOverview.tsx:32`), not validated savings.
5. **‚ö†ÔłŹ No Green-Belt mentoring / oversight link.** Reusing Green's `DMAICPhase` is structural reuse,
   not a mentoring relationship (no Black-oversees-Green-project FK anywhere).

## Feature proposals + redesign (ranked ‚ÄĒ both directions)
Ranked `weight √ó user_value √ó low_effort`:

1. **(weight‚ÜĎ val‚ÜĎ effort-med) Enforce the tollgate gate.** Add a service check: a `DMAICPhase`
   cannot move to `completed` unless its `sixsigma.TollgateReview` for that phase is `approved`/
   `passed`. Wire into the shared `DMAICPhaseViewSet.update` (used by both Green & Black) +
   `LSSBlackPhases.tsx:80`. *This converts the existing record into a real gate ‚ÄĒ highest value,
   reuses two models already present.*
2. **(weight‚ÜĎ val‚ÜĎ effort-med) Link advanced tools to the decision spine.** Add nullable FKs on
   `lss_black` models: `HypothesisTest.phase` + `HypothesisTest.root_cause` (‚Üí `sixsigma.FishboneCause`),
   `DesignOfExperiment.improves_solution` (‚Üí `sixsigma.Solution`), `SPCChart.control_plan` (‚ÜĒ
   `lss_black.ControlPlan`), `ControlPlan.spc_chart`. Surface "tie to root cause / push to control"
   buttons in `LSSBlackHypothesisTests.tsx` / `LSSBlackControlPlans.tsx`. *Carries data-guardian
   backup warning ‚ÄĒ needs migration.*
3. **(weight‚ÜĎ val-med effort-low) De-duplicate.** Pick ONE backend per concept. Recommend: retire
   `lss_black.HypothesisTest`/`SPCChart`/`ControlPlan` in favour of the richer `sixsigma` equivalents
   (regression/correlation types, FMEA, closure already there), keeping only **DOE** as genuinely
   Black-only. Remove the duplicate "Hypothesis Tests" sidebar item (`AppSidebar.tsx:449` vs `:492`).
   *Avoids the two-backends-one-concept trap; coordinate with `lss-green-methodology-aligner`.*
4. **(weight-med val‚ÜĎ effort-med) Validated-benefits ledger on Black artifacts.** Reuse
   `sixsigma.ProjectClosure` savings/ROI; add a finance sign-off FK so a Black project's claimed
   savings require approval; surface on `LSSBlackOverview.tsx` instead of generic cost-summary.
5. **(weight-med val-med effort-med) SPC special-cause rule engine.** Evaluate Nelson/Western-Electric
   rules over `SPCChart.data_points` server-side; flag out-of-control points + trigger the linked
   `ControlPlan.reaction_plan`. Closes the Control "monitoring" gap.
6. **(weight-low val-med effort-med) Green-Belt mentoring link.** Add a BB-oversees-GB-project relation
   so the Black Belt's mentoring role is modelled, not just implied by phase-model reuse.

**Coherent redesign:** keep ONE shared DMAIC spine (`sixsigma` + Green's `DMAICPhase`), make tollgate +
MSA real *gates*, and treat `lss_black/` as a **thin integration layer that connects advanced studies
(DOE, hypothesis, SPC) to the spine's root causes / solutions / control plans / benefits** ‚ÄĒ deleting
the duplicated record-stores rather than letting two parallel hypothesis/SPC/control-plan backends drift.
