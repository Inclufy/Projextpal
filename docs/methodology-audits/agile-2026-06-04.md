|---|---|---|

Overall weighted fidelity ≈ **57%**. The data model is broad and well-seeded (`views.py:563` `AgileSeedDemoView` fills every tab), but several artifacts are *stored, not enforced*, and the iteration loop has two breaks (assignment, DoD gating).

## Value/principle violations (highest priority)
1. ❌ **DoD is not a gate** — "working software = progress measure." `DefinitionOfDone` (`models.py:356`) is plain CRUD; `AgileIterationViewSet.complete` (`views.py:376-383`) sets `status='completed'` and copies `completed_points` with **zero DoD check**. Nothing prevents marking an item/iteration done with required criteria unmet. Frontend DoD page (`AgileDefinitionOfDone.tsx`) is a checklist editor with no per-item completion state.
2. ❌ **Assignment is invisible in execution** — "motivated individuals / self-organizing teams." `AgileBacklogItem.assignee` exists (`models.py:165`) and is seeded, but **no agile page reads or sets `assignee`** (grep: 0 hits in `frontend/src/pages/agile/`). The Iteration Board (`AgileIterationBoard.tsx:79-83`) renders type + points + title only — a team cannot assign work through the UI.
3. ⚠️ **Vision traces to goals but the chain dead-ends** — "early + continuous delivery of value." `AgileProductGoal.vision` FK exists (`models.py:67`) so Vision→Goal traces. But `AgileEpic` (`models.py:111`) and `AgileBacklogItem` (`models.py:156`) FK only to `Project`, **never to a goal**. So Vision→Goal→Epic→Item does NOT trace; goals are an isolated island. "Responding to change" backlog re-prioritization is `order`-only (`reorder` action, `views.py:327`) — no value/WSJF signal.
4. ⚠️ **Velocity is committed at start, not recomputed** — `start` snapshots `velocity_committed = total_points` (`views.py:370`); `complete` snapshots `velocity_completed` once (`views.py:380`). Mid-iteration scope change is not reflected; Velocity page (`AgileVelocity.tsx:22`) reads only these frozen integers.

## Duplication-vs-Scrum verdict
**Verdict: ~80% redundant Scrum clone with weaker enforcement.** `agile/` is functionally a renamed, thinner copy of `scrum/`:

| Concept | `agile/` | `scrum/` | Assessment |
|---|---|---|---|
| Backlog item | `AgileBacklogItem` (`models.py:134`) | `BacklogItem` (`scrum/models.py:18`) | duplicate; scrum adds `parent`, `reporter` |
| Timebox | `AgileIteration` (`models.py:178`) | `Sprint` (`scrum/models.py:71`) | duplicate; same status machine |
| Daily | `AgileDailyUpdate` (`models.py:251`) | `DailyStandup`+`StandupUpdate` (`scrum/models.py:309`) | duplicate |
| Retro | `AgileRetrospective`/`Item` (`models.py:270`) | `SprintRetrospective` (`scrum/models.py:403`) | duplicate |
| Velocity | frozen ints on iteration | `Velocity` model + `completion_rate` (`scrum/models.py:417`) | scrum is richer |
| DoD | `agile.DefinitionOfDone` (`models.py:356`) | `scrum.DefinitionOfDone` + `DoDChecklistEntry` (`scrum/models.py:434,530`) | **distinct tables** (`db_table='agile_definition_of_done'`), but scrum has per-sprint sign-off entries agile lacks |
| Increment/Release | `AgileRelease` (`models.py:219`) | `Increment` w/ `meets_dod`,`is_released` (`scrum/models.py:222`) | scrum models shippability; agile does not |

Agile's only genuinely distinct artifacts are **Product Vision/Goals, User Personas, and Budget** — none of which is Agile-specific (they are generic discovery/finance, also fitting Waterfall/PRINCE2). The on-disk DoD-table split (safety rule #4) is correctly maintained, so no model-name collision — but conceptual collision is total.

**Recommendation: DIFFERENTIATE, do not consolidate yet** (a live customer uses these routes). Agile should lean into the Manifesto-generic value it does NOT yet deliver: **outcome/value metrics over output**, **flexible cadence** (continuous-flow / Kanban WIP limits, not fixed sprints), and **multi-framework neutrality**. If differentiation isn't pursued next quarter, the honest call is to deprecate `agile/` and route to `scrum/`.

**Operability blockers:** (1) the board is a *viewer*, not a workboard — items cannot be assigned or transitioned in-place; status only changes via the Backlog edit dialog. (2) "Done" is unguarded. These two break a real iteration walk.

## Feature proposals + redesign (ranked — both directions)
1. **DoD gate on done** (weight High × value High × low effort). Add `AgileDodEntry(dod_item FK, iteration FK, backlog_item FK null, completed, completed_by, completed_at)` mirroring `scrum.DoDChecklistEntry` (`scrum/models.py:530`). Block `AgileIterationViewSet.complete` / item→done unless required entries pass. Page: checklist with tick state on Iteration Board card. *Closes violation #1.*
2. **In-board assignment + status transitions** (High × High × med). Make `AgileIterationBoard.tsx` a real workboard: render `assignee_name` (already in serializer, `serializers.py:83`), add an assignee `Select`, PATCH `status` on drag/drop. *Closes violation #2 + operability blockers.*
3. **Goal→Epic traceability** (Med × High × low). Add `AgileEpic.goal = FK(AgileProductGoal, null=True)`; show goal chip on Epics/Backlog and a Vision→Goal→Epic→Item rollup on Overview. *Closes violation #3.*
4. **Redesign — Manifesto-generic differentiation** (structural). Pivot agile away from fixed sprints toward **continuous flow**: add WIP limits per board column + cycle-time/throughput metrics (replace frozen velocity with computed flow metrics), and an **outcome metric** model (target vs actual on `AgileProductGoal.success_criteria`). This is the only path that makes `agile/` not-a-scrum-clone.
5. **Recompute velocity from live state** (Med × Med × low). Replace frozen `velocity_completed` reads with `iteration.completed_points` property (already exists, `models.py:207`) in serializer + Velocity page.

`--scaffold` not run (read-only). Items 1 and 3 require new migrations → **data-guardian backup gate (CLAUDE.md §9) before any `makemigrations agile` is applied** (live customer `zanjabil@inclufy.com`).
