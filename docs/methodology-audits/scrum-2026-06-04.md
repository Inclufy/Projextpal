## Pillar / commitment violations (highest priority)
- **Definition of Done — does NOT gate the Increment (breaks Transparency / "Done").** `Increment.meets_dod` is a free boolean (`models.py:251`) never set from real checks; `DoDChecklistEntry` (`models.py:530`) and `DoDChecklistCompletion` (`models.py:484`) have no UI; `BacklogItem.update_status` accepts `done` with zero DoD check (`views.py:186-193`). An item is "Done" on a dropdown click. DoD page is criteria CRUD only (`ScrumDefinitionOfDone.tsx`).
- **Product Goal — entirely absent (breaks Product Backlog commitment).** No model, no field, no page, no sidebar item (grep: 0 hits across backend/frontend). `ProductBacklog.vision` (`models.py:10`) is the closest, and it is unused by any flow.
- **Sprint Goal — stored, never committed at Planning (breaks Sprint Backlog commitment).** `SprintGoal` model + viewset + routes exist (`models.py:123`, `views.py:630`, `urls.py:195`) but **no frontend page** consumes them. The Planning page (`ScrumSprintPlanning.tsx`) never sets a goal; `Sprint.goal` is a loose text field set only at sprint create (`ScrumSprintBoard.tsx:126`).
- **Sprint Planning — does NOT select Product Backlog Items into the Sprint (breaks the Why/What/How event).** `SprintPlanning` is a meeting-log CRUD (`ScrumSprintPlanning.tsx`): no PBI picker, no commit. Selection happens incidentally on a different page (Backlog → `assign_to_sprint`), decoupled from the Planning event.
- **Sprint Review — no per-item acceptance, no real backlog adaptation.** `ScrumSprintReview.tsx` has no sprint selector, no `demo_items` picker; "Backlog Updates" is free text (`:49`) that never writes to the backlog. `sprint_goal_achieved` (`models.py:353`) is never set from the UI.
- **Burndown — recorded but never shown (breaks Inspection).** `record_burndown` computes real remaining/completed (`views.py:386-403`) but **no frontend page renders any burndown chart** (grep: 0 hits in `pages/scrum/`).
- **DoDChecklistEntry — orphaned model.** Defined `models.py:530` but absent from `serializers.py`, `views.py`, `urls.py` — dead schema.
